Research Hub — One-Time GitHub Setup

Estimated time: 15 minutes. You only do this once. After this, all future publishing is automatic.

What you are setting up

Two GitHub repos: 1. research-hub (private) — contains everything in this folder: source .qmd files, configs, the GitHub Actions workflow. This stays private indefinitely — your drafts live here. 2. drmohammadnabhan.github.io (public) — receives only the built HTML output. This is what visitors see at https://drmohammadnabhan.github.io. Before publication, it shows a minimal landing page. At publication, one push deploys the full site automatically.

A GitHub Personal Access Token (PAT) acts as a secure key that lets the private repo push its built output to the public repo. You create it once and store it as a secret.

Step 1 — Create the research-hub private repo

  1. Go to github.com/new
  2. Repository name: research-hub
  3. Visibility: Private ← important
  4. Do NOT check “Add a README file” (leave everything unchecked)
  5. Click “Create repository”

Leave this page open — you’ll need the URL in Step 5.

Step 2 — Create the drmohammadnabhan.github.io public repo

  1. Go to github.com/new
  2. Repository name: drmohammadnabhan.github.io (must be exactly this)
  3. Visibility: Public ← must be public for GitHub Pages
  4. Check “Add a README file” (so the main branch exists from the start)
  5. Click “Create repository”

Then enable Pages on this repo: 1. In the new repo, click “Settings” (top menu) 2. In the left sidebar, click “Pages” 3. Under “Source”, select “Deploy from a branch” 4. Branch: main / / (root) 5. Click “Save”

Step 3 — Create a Personal Access Token (PAT)

This is the secure key that lets research-hub push to drmohammadnabhan.github.io.

  1. Go to github.com/settings/tokens (or: GitHub → Your profile icon → Settings → Developer settings → Personal access tokens → Tokens (classic))
  2. Click “Generate new token (classic)”
  3. Note: research-hub-deploy
  4. Expiration: No expiration (or 1 year — your preference)
  5. Scopes: Check the box next to repo (this selects all repo sub-permissions)
  6. Scroll down and click “Generate token”
  7. ⚠️ Copy the token immediately — it will not be shown again. Paste it somewhere temporarily (Notes, TextEdit) while you complete the next step.

Step 4 — Add the PAT as a secret in research-hub

  1. Go to github.com/drmohammadnabhan/research-hub
  2. Click “Settings” (top menu)
  3. Left sidebar: “Secrets and variables” → “Actions”
  4. Click “New repository secret”
  5. Name: DEPLOY_TOKEN (exactly this, all caps)
  6. Secret: paste the token you copied in Step 3
  7. Click “Add secret”

You can now delete the temporary copy of the token from Notes/TextEdit.

Step 5 — Initialize git in this folder and push

Open Terminal on your Mac:

# Navigate to this folder
# (Replace the path below with the actual path to your Nabsidian vault —
#  right-click the Research_Hub folder in Finder → "Get Info" to find it,
#  or drag the folder into Terminal after typing "cd ")
cd "/path/to/your/Nabsidian 2026/Research_Hub"

# Initialize git
git init
git branch -M main

# Connect to the private repo
git remote add origin https://github.com/drmohammadnabhan/research-hub.git

# Stage everything and make the first commit
git add .
git commit -m "Initial Research Hub setup"

# Push — this triggers the first GitHub Actions build
git push -u origin main

When prompted for credentials: enter your GitHub username (drmohammadnabhan) and use your PAT (the token from Step 3) as the password. GitHub no longer accepts account passwords for git push — the PAT is the correct credential.

To avoid being prompted every time, run this once after the push:

git config credential.helper osxkeychain

macOS will store the credential securely in Keychain and you will not be prompted again.

Step 6 — Watch the first deployment

  1. Go to github.com/drmohammadnabhan/research-hub/actions
  2. You should see a workflow run called “Build and Deploy Research Hub” in progress
  3. It takes approximately 2–3 minutes (installs Quarto + Python, builds the site, pushes HTML)
  4. When it shows a green ✓, go to drmohammadnabhan.github.io
  5. Your research site is live

If the workflow shows a red ✗, click on it to see the error log. The most common issues are: wrong secret name (must be DEPLOY_TOKEN), wrong repo name in deploy.yml, or Pages not enabled on the public repo.

After Setup — Normal Publishing Workflow

Every future update is exactly three commands from the Research_Hub/ folder:

cd "/path/to/Nabsidian 2026/Research_Hub"
git add .
git commit -m "describe what you changed"
git push

GitHub Actions does the rest automatically. The site updates in ~3 minutes.

For pre-publication: the site is visible at your GitHub Pages URL, but since most of your research pages have “Under Review” notices and no raw research data, this is acceptable public presence. If you want the site completely hidden until publication, temporarily replace index.qmd with a “Coming Soon” placeholder — a future Claude session can manage this.

At publication: update the badge-preprint to badge-published, add the DOI to the citation blocks, activate the demo page, push once. Done.

Ongoing Workflow (What Claude Handles)

When starting a session to update the Research Hub:

“Update the Research Hub for the COO project. [Describe what to add/change.]”

Claude will edit the relevant .qmd files in Research_Hub/projects/coo/. You then run the three git commands above. Claude does not push to git — you review and push manually, keeping you in control of what goes public.

Troubleshooting Quick Reference

Problem Likely cause Fix
Workflow fails: Error: Resource not accessible by integration Wrong PAT scope or wrong secret name Regenerate PAT with repo scope; check secret is named DEPLOY_TOKEN
Workflow fails: remote: Repository not found Repo name mismatch in deploy.yml Check external_repository: drmohammadnabhan/drmohammadnabhan.github.io matches exactly
Pages shows “404” Pages not enabled or branch not set Repeat Step 2 Pages configuration
Math not rendering MathJax CDN blocked Reload page; check browser console
Push asks for password every time Keychain not configured Run git config credential.helper osxkeychain
