Experimental Results

CEC 2020 Engineering Benchmark Comparison

Note

Status: Full results tables and raw data will be released upon paper acceptance. This page summarizes the key findings.

Benchmark Setup

COO was evaluated on six real-world engineering design problems from the CEC 2020 Real-World Constrained Optimization Benchmark (Kumar et al., 2020). Each method was run five times from independent random feasible starting points. All methods used the same Phase-1 feasibility restoration before the main optimization, ensuring the comparison focuses on optimization performance rather than feasibility-finding.

Comparison methods:

  • COO — this work
  • PGD — Penalty Gradient Descent using L-BFGS-B on the penalty-augmented objective \(f(x) + \rho\sum_i \max(0, G_i(x))^2\)
  • PSO — Particle Swarm Optimization with 30 particles

Metrics:

  • Success Rate (SR): fraction of 5 runs achieving \(f \leq f_{\text{ref}}(1 + 0.01)\) with feasibility — within 1% of the reference optimum
  • Mean Gap: \(\overline{(f - f_{\text{ref}})/|f_{\text{ref}}|}\) across 5 runs
  • Gradient evaluations: total gradient calls used

Key Findings

P1 — Three-bar truss: efficiency advantage

All three methods achieve SR = 100%. COO uses approximately 1,500 gradient evaluations; PGD requires approximately 30,000. COO is ~20× more gradient-efficient at equal accuracy. For problems where gradient evaluations are cheap, this difference is minor. For simulation-backed design where each gradient call involves a finite-element solve or CFD run, the 20× difference is the entire budget.

P5 — Speed reducer: the flagship result

Tip

COO: SR = 100%. PGD: SR = 0%. Penalty gradient descent fails on all five runs.

The 11-constraint speed reducer is the most complex benchmark in the set. Penalty gradient descent aggregates all 11 constraints into a single combined penalty term — the resulting landscape is highly non-convex and riddled with local optima that trap L-BFGS-B. COO’s sequential Snell-KKT reflections handle each constraint boundary as an independent geometric event, decomposing the multi-constraint surface naturally and generating shadow prices for each constraint individually.

PSO also achieves SR = 100% but uses no gradient information and generates no shadow prices. For a practitioner who needs to know which of the 11 gearbox constraints is the binding bottleneck and by how much — the actionable design insight — COO provides this information and PSO does not.

P3, P4 — Honest failure analysis

Warning

COO performs poorly on the welded beam (P3, mean gap ~640%) and tension/compression spring (P4, mean gap ~124%).

These problems have narrow feasible corridors. When the feasible region is a thin valley, sequential Snell-KKT reflections trap the trajectory in a bouncing pattern between opposing walls — the algorithm cannot traverse the corridor effectively. This is an analytically understood structural limitation of the current algorithm, not a parameter sensitivity issue.

COO is designed for problems where constraints are active at the optimum and there is geometric structure to exploit at the boundary. Narrow tunnel geometry violates the Normal Approach Condition (NAC) that underlies the algorithm’s geometric efficiency. For such problems, standard gradient projection or interior-point methods are preferable.

P6 — Parameter sensitivity

COO achieves SR = 0% on P6 (Himmelblau’s function with tight constraints) under default parameters (\(\tau = 10\)). The Ombra fires too rarely and the algorithm exhausts its budget before converging to within the 1% success threshold. Reducing \(\tau\) to 5 addresses this. This is a parameter sensitivity issue, not a fundamental limitation. See Practitioner Guidelines → for parameter tuning guidance.

Cost Profile

COO uses approximately 1,500 gradient evaluations across every problem — flat and predictable regardless of difficulty. This is because the Lumen runs for a fixed iteration budget and the Ombra sub-calls use at most 300 steps. For applications with expensive gradient evaluations, a predictable and budget-bounded method is a practical advantage independent of its accuracy.

Shadow Price Quality

A result with no analogue in the comparison methods: COO generates converging KKT multiplier estimates throughout the optimization run. For P5 (speed reducer), the Luminosity Flux at convergence identifies the binding gearbox constraint — the one limiting the design — and provides its shadow price (sensitivity to relaxation). This information guides the next design iteration. Neither PGD nor PSO generates it.

Theorem 5 establishes \(\|\bar\Lambda_k - \lambda^*\| = O(1/k^2)\) for the averaged Luminosity Flux. The individual multipliers at each boundary event decay to zero as the orbit converges (see Remark 6 in the paper) — the averaging is not merely convenient but necessary to recover the true shadow prices.
